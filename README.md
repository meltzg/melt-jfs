# melt-jfs

A Java NIO `FileSystemProvider` implementation for MTP (Media Transfer Protocol) devices such as Android phones and dedicated audio players. Uses the Java Foreign Function & Memory (FFM) API (Project Panama) instead of JNI, so no native build step is required.

## Requirements

- **Java 21** (FFM is a preview feature; `--enable-preview` is set automatically by the build)
- **Gradle** — use the included wrapper (`./gradlew`); no separate Gradle installation needed

## System libraries (Linux)

The runtime library is loaded at runtime via FFM; no compile-time linking step is needed. Install the shared libraries with:

```bash
sudo apt install libmtp9 libglib2.0-0
```

| Library | Package | Purpose |
|---------|---------|---------|
| `libmtp.so.9` | `libmtp9` | MTP device communication |
| `libgobject-2.0.so.0` | `libglib2.0-0` | GObject (required by libumockdev at test time) |

## Building

```bash
./gradlew build
```

## Running tests

Tests that require a real or mocked MTP device are skipped automatically when the required libraries or fixture files are absent, so a plain `./gradlew test` always succeeds in CI.

### Unit tests (no device needed)

```bash
./gradlew test
```

### Device-mocked integration tests

The integration tests use [umockdev](https://github.com/martinpitt/umockdev) to replay recorded USB traffic without a physical device. Two things are needed:

1. **umockdev runtime libraries** — provides `libumockdev-preload.so.0`, which intercepts `open()` calls so `libusb` sees the fake device tree:

   ```bash
   sudo apt install umockdev
   ```

2. **Fixture files** — recorded USB traffic from a real device. Generate them once with a connected device, then commit and reuse in CI:

   ```bash
   # Connect the MTP device, then:
   ./record-fixture.sh
   # Follow the prompts — run ./gradlew test in a second terminal when asked
   git add src/test/resources/umockdev/
   git commit -m "Add umockdev fixtures"
   ```

   The tests skip gracefully if `src/test/resources/umockdev/ak100ii.umockdev` or `ak100ii.ioctl` are missing.

### Full dev dependency list

For a machine that will both run tests and record new fixtures:

```bash
sudo apt install libmtp9 libmtp-dev libglib2.0-0 libumockdev0 libumockdev-dev umockdev
```

| Package | Needed for |
|---------|-----------|
| `libmtp9` | Runtime — MTP device access |
| `libmtp-dev` | Optional — headers only, not required by the FFM build |
| `libglib2.0-0` | Runtime — GLib/GObject (dependency of libumockdev) |
| `libumockdev0` | Test runtime — `libumockdev.so.0` loaded by `UMockdevTestbed` |
| `libumockdev-dev` | Optional — headers only |
| `umockdev` | Recording and test preload — `umockdev-record`, `libumockdev-preload.so.0` |

## Project structure

```
src/
  main/java/org/meltzg/fs/mtp/
    MTPFileSystemProvider.java   # NIO SPI entry point
    MTPFileSystem.java           # FileSystem implementation
    MTPPath.java                 # Path implementation
    MTPFileStore.java            # FileStore (per storage volume)
    MTPBasicFileAttributes.java  # BasicFileAttributes wrapper
    MTPDeviceBridge.java         # Thread-safe device access singleton
    LibMTP.java                  # FFM bindings for libmtp
    types/                       # Value types (Lombok @Value)
  test/java/org/meltzg/fs/mtp/
    MTPDeviceBridgeTest.java     # Integration test (umockdev-based)
    UMockdevTestbed.java         # FFM wrapper for libumockdev
  test/resources/umockdev/       # Fixture files (generated by record-fixture.sh)
record-fixture.sh                # Helper script to record device fixtures
```

## Platform notes

The FFM struct layouts in `LibMTP.java` are defined for **x86-64 Linux** against **libmtp 1.1.x**. For other architectures or libmtp versions, regenerate the layouts with [jextract](https://github.com/openjdk/jextract) against the installed `libmtp.h`.
